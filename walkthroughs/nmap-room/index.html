This walkthrough covers the full TryHackMe Nmap room and is perfect for beginners learning how to scan, enumerate, and identify services using nmap. You'll learn about different scan types like TCP, SYN, UDP, as well as using Nmap Scripting Engine (NSE), and techniques to evade firewalls.

By the end, you’ll be confident in:

  • Using common nmap switches like -sS, -sV, -O, -p-
  • Running different scan types for stealth or reliability
  • Understanding firewall evasion techniques using -D, --data-length, etc.
  • Working with NSE scripts to discover vulnerabilities and gather intel

Task 2: Introduction

An overview of what Nmap is, its role in reconnaissance, and why it's a core tool for network scanning and enumeration.

1. What does Nmap stand for?
Ans: Network Mapper

2. What year was Nmap released?
Ans: 1997

3. Who is the author of Nmap?
Ans: Gordon Lyon

4. Which operating system did Nmap first support?
Ans: Linux

5. What language is Nmap written in?
Ans: C, C++

Task 3: Nmap Switches

Understand various command-line switches in Nmap used for controlling the type and behavior of scans.

1. What is the switch for a SYN scan?
Ans: -sS

2. What is the switch for a UDP scan?
Ans: -sU

3. What is the switch for a service scan?
Ans: -sV

4. What is the switch to scan all ports?
Ans: -p-

5. What switch should you include to detect the operating system?
Ans: -O

Task 4: Scan Types (Overview)

Introduces different Nmap scan types like TCP Connect, SYN, NULL, FIN, Xmas, and UDP — each suited for different reconnaissance situations.

1. Which scan type uses a full TCP handshake?
Ans: TCP Connect (-sT)

2. Which scan type is known as a half-open scan?
Ans: SYN Scan (-sS)

Task 5: TCP Connect Scans

Learn about TCP Connect scans, which are reliable but easily detected by firewalls due to full handshake usage.

1. What switch is used for a TCP Connect scan?
Ans: -sT

Task 6: SYN Scans

Understand SYN scans, which send a single SYN packet and determine if a port is open based on response without completing the handshake.

1. What switch is used for a SYN scan?
Ans: -sS

Task 7: UDP Scans

Explore how UDP scans work differently from TCP and why they’re slower and trickier due to the stateless nature of UDP.

1. What switch is used for a UDP scan?
Ans: -sU

Task 8: NULL, FIN, Xmas Scans

These scans are designed to bypass poorly configured firewalls by sending malformed packets.

1. What switch sends a NULL scan?
Ans: -sN

2. What switch sends a FIN scan?
Ans: -sF

3. What switch sends a Xmas scan?
Ans: -sX

Task 9: ICMP Network Scanning

ICMP scans test if a host is up by sending echo requests and checking if there's a response.

1. What switch disables DNS resolution?
Ans: -n

2. What switch will tell Nmap to use ICMP echo requests only?
Ans: -PE

Task 10: NSE Overview

Nmap Scripting Engine (NSE) allows powerful extensibility by using Lua scripts to automate scanning, detection, and exploitation tasks.

1. What scripting language does NSE use?
Ans: Lua

Task 11: Working with NSE

Use scripts from default, discovery, safe, or vuln categories to dig deeper into target services.

1. What category of scripts is considered safe to run?
Ans: safe

Task 12: Searching for Scripts

Use `locate` or `find` to identify NSE scripts on your system, or search by keywords using `grep`.

1. What command would you use to find scripts with the word "ftp" in them?
Ans: locate *.nse | grep ftp

Task 13: Firewall Evasion

Use techniques like spoofed MACs, decoy IPs, or fragmentation to bypass restrictive firewalls.

1. What switch would you use to set a decoy?
Ans: -D

2. What switch allows you to set a specific MAC address?

3. What switch would you use for data length modification?
Ans: --data-length
