SYN (stealth/half-open) scan ‚Äî default, fast & quiet:

üìö Service & Version Detection

Detect running services and versions:

Scan specific ports:

üß† NSE Scripts (Nmap Scripting Engine)

NSE scripts can detect CVEs, brute-force login pages, scan web directories, and more. They're grouped into categories like auth, vuln, safe, discovery.

Run default safe scripts:

nmap -sC target_ip

Run a category of scripts:

nmap --script vuln target_ip

Run multiple scripts:

nmap --script=ftp-anon,http-title target_ip

Run all scripts matching a keyword:

nmap --script 'http-*' target_ip

Increase verbosity for NSE results:

nmap --script vuln -vv target_ip

üõ°Ô∏è Firewall Evasion

Sometimes stealth matters more than speed. These options help you bypass filters, confuse IDS/IPS, or avoid triggering alerts.

Decoy scan (spoof multiple IPs):

nmap -sS -D RND:10 target_ip

Spoof MAC address to impersonate hardware vendors:

nmap --spoof-mac Apple target_ip

Send fragmented packets (evade deep packet inspection):

nmap -f target_ip

Randomize scan timing:

nmap -T1 target_ip  # T0 to T5 (T5 = fastest, T0 = sneakiest)

Custom payload size (defeat size-based filters):

nmap --data-length 42 target_ip

Use specific source port (bypass rules expecting DNS, HTTP):

nmap --source-port 53 target_ip

üß© Host Discovery

Before scanning ports, use these to check if targets are alive or filter targets more aggressively.

Ping sweep of a subnet:

nmap -sn 192.168.1.0/24

Skip host discovery (treat all targets as alive):

nmap -Pn target_ip

TCP SYN ping to specific ports:

nmap -PS22,80 target_ip

ICMP echo (ping):

nmap -PE target_ip

ARP discovery (best for local network):

nmap -PR 192.168.1.0/24

üìù Output & Reporting

Save and reuse your scan results with these output options:

Normal output:

XML output (good for automation tools):

Grepable output:

nmap -oG output.gnmap target_ip

All at once:

nmap -oA scan_results target_ip

üí£ Pro Scan Combos

These are real-world scan combos to deploy in labs, pentests, or CTF recon:

Full TCP, all ports, with scripts & OS detect:

nmap -sS -p- -sV -sC -O -T4 -oA full target

UDP scan with service detection and top ports:

nmap -sU --top-ports 200 -sV target

Top 100 TCP ports, fast, no DNS:

nmap --top-ports 100 -T4 -n target

Web-targeted scan:

nmap -p 80,443,8080 --script=http-title,http-methods -sV target

Stealthy: slow timing, decoy IPs, custom source port:

nmap -sS -D RND:5 -T1 --source-port 53 target

üöÄ Scan Timing & Speed Control

Fine-tune performance with these timing flags:

Timing templates:

-T0  # Paranoid (ultra slow, stealthy)
-T1  # Sneaky
-T2  # Polite
-T3  # Normal (default)
-T4  # Aggressive (for LANs/CTFs)
-T5  # Insane (unstable)

Control scan speed manually:

--min-rate 1000    # Send minimum 1000 packets/sec
--max-retries 2    # Fail fast if no response

Scan all TCP + run default scripts + service/OS detection:

nmap -sS -sV -O -p- -T4 -v target_ip

Find vulnerabilities with NSE:

nmap -sV --script vuln target_ip

Web server analysis:

nmap -p 80,443 --script http-title,http-methods target_ip

üîê Web & Auth NSE Scripts

When you're scanning web servers or login services, NSE provides powerful automation:

Detect open directories or admin panels:

nmap --script http-enum -p80,443 target

Check HTTP methods allowed (GET, POST, PUT, etc):

nmap --script http-methods -p80,443 target

Run web vulnerability checks:

nmap --script http-vuln* -p80,443 target

Brute-force FTP login (if anonymous fails):

nmap --script ftp-anon,ftp-brute -p21 target

SMB enumeration (users, shares, OS):

nmap --script smb-enum-shares,smb-enum-users -p445 target

üåê IPv6 & Advanced Targeting

Scan IPv6 addresses:

Scan from a file:

nmap -iL targets.txt

Exclude hosts from a scan:

nmap target_subnet --exclude 192.168.1.1,192.168.1.254
